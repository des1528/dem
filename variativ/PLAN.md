# План: Вариативная часть — Руководство пользователя

## Контекст
Модуль 3 ДЭ-2026 «Документирование ПО». Требуется приложить к разработанному приложению («Салон красоты» из `salon/`) **руководство пользователя** в формате Word, оформленное по ГОСТ.

Это **не код**, а текстовый документ. Но чтобы оформить его быстро и без ошибок, я сгенерировал его скриптом на `python-docx` — это удобно: меняешь текст в одном файле и пересобираешь.

## Требования (из задания)
- Раздел «Основные сведения» с описанием продукта.
- Перечень ролей пользователей.
- Подробное описание функционала с иллюстрациями (скриншотами).
- Соответствие ГОСТ, единый стиль оформления.
- Стили заголовков, автоматическое оглавление.
- Без жаргонизмов и опечаток.

## Структура документа
1. Титульный лист (по центру, крупно).
2. Автоматическое оглавление (поле TOC).
3. **1. Основные сведения о продукте**
   - 1.1. Роли пользователей (таблица: роль → возможности).
4. **2. Установка и запуск** (требования к ПО, команда запуска).
5. **3. Работа с системой** — по разделам:
   - 3.1. Главное окно (+ Рисунок 1)
   - 3.2. Поиск, фильтрация, сортировка
   - 3.3. Вход в режим администратора (+ Рисунок 2)
   - 3.4. Добавление услуги
   - 3.5. Изменение услуги
   - 3.6. Удаление услуги
   - 3.7. Запись клиента (+ Рисунок 3)
   - 3.8. Ближайшие записи
6. **4. Сообщения об ошибках** (таблица: сообщение → причина).

## Подготовка
```
pip install python-docx
python build_userguide.py
```

После генерации:
1. Открыть в Word.
2. Нажать F9 на оглавлении → оно подтянется автоматически.
3. Сделать 3 скриншота приложения и вставить вместо подписей «(Рисунок 1 — ...)».
4. Сохранить.
