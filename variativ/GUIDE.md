# GUIDE — как написать Руководство пользователя на экзамене

Это **не код**. Это документ Word, который оценивается по ГОСТ-критериям. На экзамене у тебя два пути: **писать вручную** в Word или **сгенерировать через python-docx**. Я рекомендую второй — потому что генератор гарантирует единый стиль, нумерацию и таблицы.

---

## 1. Критерии оценки (из ТЗ)

- Раздел с основными сведениями присутствует.
- Описаны роли пользователя.
- Нет жаргонизмов и ошибок.
- Документ соответствует ГОСТ.
- Единый стиль оформления.
- Оформление через стили (не ручное форматирование).
- Автоматическое оглавление, гиперссылки на таблицы и рисунки.
- Все необходимые разделы присутствуют.
- Описан функционал всех ролей.
- Есть иллюстрации (скриншоты).
- Документ понятен пользователю без IT-образования.

---

## 2. Обязательная структура

1. **Титульный лист** — название документа, название системы, год.
2. **Оглавление** — автоматическое, через поле TOC.
3. **1. Основные сведения** — что это за продукт, какую задачу решает.
   - **1.1. Роли пользователей** — таблица «роль / возможности».
4. **2. Установка и запуск** — требования к ПО, команда.
5. **3. Работа с системой** — пошагово по основным сценариям. Каждый сценарий — отдельный подраздел + скриншот.
6. **4. Сообщения об ошибках** — таблица «сообщение / причина и устранение».

## 3. Генерация через `python-docx`

### 3.1 Базовые приёмы

```python
from docx import Document
from docx.shared import Pt, RGBColor
doc = Document()
doc.styles['Normal'].font.name = "Times New Roman"
doc.styles['Normal'].font.size = Pt(14)

doc.add_heading("1. Основные сведения", level=1)
doc.add_paragraph("Длинный абзац текста.")
doc.add_paragraph("маркер", style="List Bullet")
```

`add_heading` со стилем `Heading 1/2` — автоматически попадёт в оглавление. Это и есть «оформление через стили».

### 3.2 Таблица

```python
t = doc.add_table(rows=1, cols=2); t.style = "Light Grid"
t.rows[0].cells[0].text = "Колонка 1"
t.rows[0].cells[1].text = "Колонка 2"
r = t.add_row().cells; r[0].text = "..."; r[1].text = "..."
```

### 3.3 Автоматическое оглавление

`python-docx` не умеет вставлять TOC напрямую — нужно через низкоуровневое OOXML:

```python
from docx.oxml import OxmlElement
from docx.oxml.ns import qn

def add_toc(doc):
    p = doc.add_paragraph(); run = p.add_run()
    fldChar1 = OxmlElement('w:fldChar'); fldChar1.set(qn('w:fldCharType'),'begin')
    instr = OxmlElement('w:instrText'); instr.set(qn('xml:space'),'preserve')
    instr.text = 'TOC \\o "1-3" \\h \\z \\u'
    fldChar2 = OxmlElement('w:fldChar'); fldChar2.set(qn('w:fldCharType'),'separate')
    text = OxmlElement('w:t'); text.text = "Оглавление обновите по F9"
    fldChar3 = OxmlElement('w:fldChar'); fldChar3.set(qn('w:fldCharType'),'end')
    for el in (fldChar1, instr, fldChar2, text, fldChar3):
        run._r.append(el)
```

В Word'е после открытия документа жмёшь F9 на оглавлении — оно подтягивается.

## 4. Чего НЕ должно быть

| Плохо | Лучше |
|---|---|
| «Тыкаешь кнопочку» | «Нажмите кнопку» |
| «Жмёшь Enter и оно отрисует» | «После нажатия клавиши Enter список обновляется» |
| «Чувак, ну ты понял» | официальный нейтральный тон |
| «Эта приблуда» | «Данная подсистема» |
| Использовать «вы»/«Вы» хаотично | выбрать одну форму и держаться её |

Главное правило: **читатель — не программист**. Не пиши «БД», пиши «база данных». Не пиши «GUI», пиши «графический интерфейс».

## 5. Скриншоты

Минимум 3 скриншота:
1. Главное окно со списком услуг.
2. Окно ввода кода администратора.
3. Окно записи клиента.

В Windows: `Win+Shift+S` → выделить область → Ctrl+V в Word на место подписи «(Рисунок N — ...)».

После вставки выделить картинку → правой кнопкой «Вставить название» → выбрать «Рисунок» → нумерация подхватится автоматически и оглавление сможет сослаться.

## 7. Шпаргалка к экзамену

Если на экзамене нельзя ставить пакеты — пишешь руками в Word:

1. Открой Word → Файл → Создать.
2. Главная → Стили → Times New Roman 14.
3. Введи заголовки через стиль **Заголовок 1** и **Заголовок 2** — НЕ просто крупным жирным.
4. Ссылки → **Оглавление** → Автособираемое.
5. Вставь все разделы из структуры выше.
6. Сделай скриншоты (Win+Shift+S) и вставь их.
7. После каждого скриншота: Ссылки → **Вставить название** → «Рисунок».
8. В конце ещё раз обнови оглавление (правый клик → «Обновить поле»).
9. Сохрани как `.docx`.
