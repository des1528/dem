Архитектурно — то же, что shoes/salon. Уникальные части: **captcha**, **дополнительные товары**, **библиотека + тесты**. Их разбираем подробно.

## 1. Captcha — пошагово

ТЗ: «4 символа (буквы латиницы или цифры), не в одной линии, перечёркнуты или наложены, плюс графический шум».

### 1.1 Генерация текста

```python
import random, string
text = "".join(random.choices(string.ascii_uppercase + string.digits, k=4))
```

Это самая простая часть. Запомни: `random.choices` (не `choice`!) даёт сразу несколько символов.

### 1.2 Рисование

```python
from PIL import Image, ImageDraw, ImageFont
img = Image.new("RGB", (160, 60), "white")
d = ImageDraw.Draw(img)
```

**Шум** — много случайных точек и пара линий:
```python
for _ in range(80): d.point((random.randint(0,160), random.randint(0,60)), fill=(rgb))
for _ in range(6):  d.line((x1,y1,x2,y2), fill=(rgb), width=1)
```

**Символы со сдвигом и поворотом** — главный приём: рисуем каждый символ на отдельном прозрачном слое, поворачиваем его и накладываем со случайным смещением:

```python
for i, ch in enumerate(text):
    ci = Image.new("RGBA", (40,50), (255,255,255,0))
    ImageDraw.Draw(ci).text((4,4), ch, font=ImageFont.truetype("arial.ttf", 32), fill=(...))
    ci = ci.rotate(random.randint(-25,25), expand=True)
    img.paste(ci, (10 + i*35 + random.randint(-3,3), random.randint(-2,8)), ci)
```

Третий аргумент `paste` — маска, для прозрачности.

**Перечёркивание** — одна горизонтальная линия поверх всего:
```python
d.line((0, random.randint(20,40), 160, random.randint(20,40)), fill="black", width=2)
```

### 1.3 Логика блокировки

Глобальные переменные:
```python
fail_count = 0
blocked_until = 0   # timestamp
```

Алгоритм:
1. Если `time.time() < blocked_until` → сообщить «ждите N секунд».
2. Если captcha видна — проверить её. Не совпало → `blocked_until = time.time() + 10`, обновить captcha, выйти.
3. Запрос в БД на логин/пароль. Если не нашли:
   - `fail_count += 1`,
   - если это была первая неудача без captcha — показать captcha.

**Запомни мнемонику:** «1 фейл = captcha; фейл с captcha = 10 секунд бан».

## 2. Дополнительные товары (AttachedProduct)

Это связь self-many-to-many (товар → товары). Таблица:
```sql
AttachedProduct(MainProductID, AttachedProductID)
```
оба FK на `Product`. Это **3НФ-приём** для самоссылок — нельзя хранить «список ID через запятую» в самом `Product`.

### Правила удаления
```python
# 1. сам товар в заказе?
if db.fetch("SELECT 1 FROM OrderItem WHERE ProductID=%s",(pid,)):
    return "запрет"
# 2. кто-то из дополнительных в заказе?
attached = [r["AttachedProductID"] for r in db.fetch("... WHERE MainProductID=%s",(pid,))]
for a in attached:
    if db.fetch("SELECT 1 FROM OrderItem WHERE ProductID=%s",(a,)):
        return "запрет"
# 3. иначе — удалить связки, дополнительные, основной
```

## 3. Библиотека `SF2022User01Lib`

### 3.1 Именование строго по ТЗ

| Что | Значение |
|---|---|
| Файл/пакет | `SF2022User{NN}Lib` (NN = номер участника) |
| Класс | `Calculations` |
| Метод | `available_periods` (snake_case для Python) |
| Параметры | `start_times: list[time], durations: list[int], begin_working_time: time, end_working_time: time, consultation_time: int` |
| Возврат | `list[str]` формата `"HH:MM-HH:MM"` |

**Не переименовывай!** ТЗ прямо предупреждает: при ошибке в именах работа не проверяется.

### 3.2 Алгоритм

1. Проверки: длины списков совпадают, `consultation_time > 0`, рабочий день не пуст.
2. Перевести `time` в `datetime(1970,1,1, hh, mm)` для арифметики.
3. Отсортировать занятые окна по началу.
4. Пройти от начала дня к концу, собирая свободные промежутки:
   ```
   cursor = day_start
   for (b_start, b_end) in busy:
       if b_start > cursor: free.append((cursor, b_start))
       cursor = max(cursor, b_end)
   if cursor < day_end: free.append((cursor, day_end))
   ```
5. Нарезать каждый `(fs, fe)` на куски `consultation_time`:
   ```
   t = fs
   while t + step <= fe:
       out.append(f"{t:%H:%M}-{(t+step):%H:%M}")
       t += step
   ```

### 3.3 Почему `datetime`, а не `time`?

`datetime.time` не поддерживает арифметику (нельзя `t + timedelta`). Поэтому мы временно превращаем в `datetime(1970,1,1, hh, mm)` — дата не важна, важно, что объект умеет складываться.

## 4. Тесты (`test_lib.py`)

Стандартный `unittest`. **TDD** значит: тест пишется первым, видим красный → пишем код → видим зелёный. Так и устроен `setUp`:

```python
def setUp(self):
    self.calc = Calculations()
```

Каждый тест → один метод `test_xxx`. 10 сценариев покрывают граничные случаи: пустой ввод, полная загрузка, отрицательные значения, разные длины массивов, неотсортированные входы.

Запуск: `python -m unittest test_lib -v`.
