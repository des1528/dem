# План: Товары для животных

## Контекст
Полная версия задания по ДЭ-2026: авторизация + captcha, список товаров с поиском/сортировкой/фильтром, CRUD товаров с дополнительными товарами, библиотека `SF2022User01Lib`, 10 unit-тестов.

## Структура
```
pets/
├── schema.sql              # БД в 3НФ
├── db.py                   # fetch/query
├── main.py                 # окно входа с captcha + главное + редактирование
├── SF2022User01Lib.py      # отдельная библиотека Calculations.available_periods
├── test_lib.py             # 10 unit-тестов
├── import_data.py
├── images/, picture.png    # placeholder
├── PLAN.md, GUIDE.md, README.md
```

## Captcha (по ТЗ)
- После **первой** неуспешной попытки — показать капчу из 4 символов (буквы латиницы + цифры), графический шум + перечёркивание/наложение.
- После неудачного входа **с** captcha — блокировка на 10 секунд.
- Реализация: рисуем `PIL.Image` 160×60, выводим символы со случайным сдвигом и поворотом (`Image.rotate`), потом проводим линию через всю картинку. Сохраняем сгенерированный текст и сравниваем с введённым (case-insensitive).

## Удаление товара (по ТЗ)
1. Если товар сам в заказе → запрет.
2. Если у товара есть дополнительные и **любой** из них в заказе → запрет.
3. Иначе удаляем сначала связки `AttachedProduct`, потом дополнительные товары, потом основной. Картинки физически удаляем.

## Изображения
- Поле `ImagePath` хранит **путь** к файлу.
- При сохранении: `Image.thumbnail((300,200))` + копирование в `images/`.
- При замене — старый файл удаляется.
- При отсутствии — `picture.png` placeholder.

## Библиотека SF2022User01Lib
- `class Calculations` с методом `available_periods(start_times, durations, begin_working_time, end_working_time, consultation_time) -> list[str]`.
- Алгоритм: сортируем занятые → находим свободные промежутки → нарезаем каждый на куски длиной `consultation_time`.
- Имена строго по ТЗ для Python (snake_case).

## Тесты (test_lib.py, 10 шт.)
1. Пример из ТЗ.
2. Пустое расписание.
3. Консультация длиннее рабочего дня.
4. Полностью занятый день.
5. Свободное окно только утром.
6. Свободное окно только вечером.
7. Слишком короткий промежуток.
8. Отрицательная длительность → ValueError.
9. Разные длины списков → ValueError.
10. Неотсортированные входные данные.

## Проверка
1. `pip install mysql-connector-python pillow openpyxl`
2. `schema.sql` в Workbench.
3. `python import_data.py`.
4. `python -m unittest test_lib -v` — должно быть 10/10 PASS.
